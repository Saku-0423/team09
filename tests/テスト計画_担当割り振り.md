# テスト計画：担当割り振り

## 目的
Magic Shooting の完成版について、3人で分担してテストを行う。
正常系（ルール通りの操作）だけでなく、異常系（想定外の操作・タイミング・連打など）も
必ず行うこと。

## 担当分け

| 担当 | 記録ファイル | テスト範囲 |
|---|---|---|
| 担当A | テスト記録01_画面遷移とボス戦.md | タイトル画面 → ステージ開始（READY） → プレイ中の敵撃破 → WARNING → ボス戦 → ステージクリア → 次ステージ／ゲームオーバーまでの一連のシーン遷移全般 |
| 担当B | テスト記録02_プレイヤー操作と武器アイテム.md | 自機の移動・発射、属性切り替え（1/2/3/スペースキー）、武器の変化（pistol/rifle/shotgun）、被弾・無敵、各種アイテム（回復・無敵・浮遊アイテム） |
| 担当C | テスト記録03_敵とUIと動作環境.md | 敵（fireSpirit/iceGolem/windSpirit）の挙動、UI表示（HP・ライフ・ENEMY数・弱点表示）、ウィンドウサイズ変更やPCスペックによる表示崩れ・重さなどの非機能面 |

## 記録のルール

- テストを行ったら、必ず配布したフォーマット通りに記録を残すこと。
- 見出しの末尾には結果に応じて `（OK）` または `（不具合あり）` を記入する。
- 「異常系」とは、以下のような “ルール通りではない使い方” を指す。
  - 連打・高速クリックなど過剰な操作
  - 画面遷移の途中でのクリック（多重クリック）
  - ウィンドウの最小化・最大化・リサイズ中の操作
  - ありえない位置へのマウス移動（画面外に出す、など）
  - 想定していないキーの入力
  - 通信/読み込みが遅い状況（PCスペックが低い環境）を想定した動作
- 不具合を見つけた場合は「備考」に再現手順・再現頻度（毎回／たまに）も書いておくこと。

## 進め方の目安

1. まずは各自、担当範囲を一通りルール通りに触ってみる（正常系）。
2. 次に、あえてルールを破った操作・タイミングをずらした操作を行う（異常系）。
3. 気になった操作は再現するかもう一度試し、再現性の有無も記録する。
4. 見た目のズレ（文字の重なり・画像のはみ出しなど）も不具合として記録する。
